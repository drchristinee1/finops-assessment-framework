# FinOps Assessment Framework

A structured approach to evaluating cloud cost governance, financial accountability, and optimization maturity.

## Purpose

This framework helps organizations understand:

- where cloud spend is coming from  
- who owns it  
- how efficiently it is used  
- and how well it aligns with business goals  

## Assessment Areas

- Cost visibility & allocation  
- Governance & accountability  
- Usage efficiency (compute, storage, database)  
- Application rationalization  
- Commitment strategy (RIs / Savings Plans)  
- Forecasting & financial planning  

## Maturity Model

Level 1: No visibility  
Level 2: Basic reporting  
Level 3: Optimization practices  
Level 4: Economic modeling  
Level 5: Decision-driven FinOps (closed-loop system)  

## Output

- Findings report  
- Cost driver analysis  
- Optimization opportunities  
- Ownership gaps  
- Strategic recommendations  

## Role in FinOps Operating System

- Detection → reveal-finops-lab  
- Action → finops-action-engine  
- Economics → driver-based-finops-modeling-engine  
- Strategy → finops-operating-system  
- Assessment → finops-assessment-framework  

## Goal

Move organizations from reactive cost tracking to proactive financial governance and decision-making.
